Deep CFR Solver

Counterfactual Regret Minimization with neural network approximation — solving imperfect information games and extending to asymmetric multi-player settings.

Project Overview

What is Deep CFR?

Classic CFR stores a regret value for every possible game state — 10¹⁶⁸ states for Texas Hold'em. Deep CFR replaces that lookup table with neural networks, enabling it to scale to previously unsolvable games.

Network Type
Deep CFR
Advantage + Strategy nets per player
Architecture
Two networks per player:
→ Advantage Net — approximates counterfactual regrets. Input: info-set vector. Output: one value per action.
→ Strategy Net — approximates average strategy (Nash output). Input: info-set vector. Output: softmax probability over actions.
Key Insight
The Nash Equilibrium is the time-average of all strategies played across iterations — not the final strategy. The strategy network accumulates this average, while the advantage network drives improvement each iteration.

How It Works

The Algorithm

Deep CFR alternates between game traversal and network training each iteration.

CFR Core Loop
1

Traverse the game tree

For the traversing player, explore every action. For other players, sample one action according to the current advantage network (external sampling MCCFR). This reduces variance while maintaining unbiased gradient estimates.

2

Compute counterfactual regrets

At each info set, calculate how much better each action would have been versus the weighted average. The counterfactual reach probability weights the regret by how often opponents would reach this node.

R(I, a) += π₋ᵢ × ( v(a, I) − v̄(I) )
3

Store in reservoir buffer

Each (info_set_tensor, regrets, iteration) tuple is added to a ReservoirBuffer. Reservoir sampling ensures that samples from ALL iterations are uniformly represented — preventing recency bias that would cause the network to forget early strategy improvements.

4

Train advantage network

Sample a batch from the reservoir. Train the advantage net to minimize weighted MSE against the stored regrets. Later iterations carry higher weights (proportional to iteration number) because they reflect more accurate estimates.

5

Regret matching → strategy

At inference, convert advantage network outputs to a probability distribution: clip negative advantages to zero, then normalize.

σ(a) = max(0, A(a)) / Σ max(0, A(a'))
6

Train strategy network

Also train the strategy network to predict the average strategy across all iterations. This is the actual Nash equilibrium output — the exploitability measure is computed against this network, not the advantage network.

Reservoir Buffer

When the buffer is full, new items replace old ones with probability capacity / total_seen. This maintains a uniform random sample from the entire history of training data — a key requirement for unbiased neural network training in CFR. 

if len(buffer) < capacity:
    buffer.append(item)
else:
    idx = random(0, total_seen)
    if idx < capacity:
        buffer[idx] = item  # reservoir!
Asymmetric Extension

For our 3-player auction game, each player has a different action space: the seller chooses a price (5 options), buyers choose accept/reject (2 options). The networks are player-conditioned via one-hot encoding of the player ID concatenated to the state vector.

input = concat( state_vec, one_hot(player_id) )

Novel Extension

Asymmetric 3-Player Auction

Our custom game: one Seller posts a price, two Buyers decide to accept or reject based on private valuations. Different action spaces per player type.

Seller's Goal
Maximize revenue = #sales × (price − cost). Has private production cost (1–3). Sets price from {2, 4, 6, 8, 10}. Must account for buyer valuations when pricing.
Buyer A
Maximizes surplus = value − price (if bought). Private value drawn from [3, 10]. Accepts only if value ≥ price. Strategic because buyer B's decision may affect seller's future pricing if games are repeated.
Buyer B
Same structure as Buyer A but acts after seeing Buyer A's decision. This sequential structure creates an information asymmetry — Buyer B observes more history.
Game Theory Concept: Mechanism Design

This game tests whether CFR can discover the revenue-maximizing mechanism: the seller should post a price just below the expected buyer valuation. At Nash Equilibrium, the seller balances higher price (more revenue per sale) vs lower acceptance rates.

E[revenue] = P(accept) × (price − cost)
P(accept) = P(v ≥ price) for each buyer
