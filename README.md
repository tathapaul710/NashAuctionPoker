# Deep CFR Experimentation & Dashboard

This repository contains a Python-based engine and a graphical web dashboard for training and analyzing AI agents using Counterfactual Regret Minimization (CFR) and Neural Networks (Deep CFR). 

The project natively solves imperfect information games like **Leduc Hold'em** and a novel **Asymmetric 3-Player Auction Game**.

## Features

- **Vanilla CFR (`cfr/vanilla_cfr.py`)**: Standard tabular CFR, evaluating exact Nash Equilibria exploits perfectly.
- **Deep CFR (`cfr/deep_cfr.py`)**: Neural Network application of CFR allowing generalization; trains an Advantage network for regrets and a Strategy network for behavioral probabilities.
- **Game Engines (`game_engine/games.py`)**: Abstractions modeling state action trees for Leduc and Custom Auctions.
- **Web Dashboard (`dashboard.html`)**: Interactive vanilla HTML/CSS/JS frontend to play against the AI and analyze the outputs generated by algorithms visually.
- **Flask Integration (`server.py`)**: A PyTorch backend server mediating frontend dashboard logic to communicate with deep neural strategy files (`.pth`) natively.

## Setup & Installation

Make sure you have Python 3.9+ installed and run:

```bash
pip install -r requirements.txt
```

## Usage

### 1. Training the Models & Generating Metrics

To initiate training on all games, run the main orchestration script:

```bash
python main.py
```

*Note: For a faster demonstration loop utilizing fewer iterations, use the `--quick` flag:*
```bash
python main.py --quick
```

Running `main.py` will:
- Evaluate Tabular CFR bounds.
- Train PyTorch Deep CFR nets on Leduc and Auction environments.
- Generate and save `results.json` mapping out returns and metrics.
- Persist matplotlib charts (such as `exploitability.png` and `leduc_loss.png`) for manual analysis.
- Save the final aggregated strategy output into `leduc_model.pth`.

### 2. Playing Against the PyTorch Agent

To use the actual Neural Network against human inputs through the visual dashboard, start the backend Flask API that links the two endpoints.

```bash
python server.py
```

This launches a microservice on `http://127.0.0.1:5000` executing the models.
Once running, simply double-click and open `dashboard.html` inside your favorite web browser! When the AI plays via the frontend, it relies exclusively on your backend machine learning variables dynamically interacting via async Javascript!
